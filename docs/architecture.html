SkillBuilder · architecture & module map

Deep modules, one seam, ports & adapters

A hexagonal / DDD scaffold of the app in ARCHITECTURE.md. Pure domain modules depend only on ports (interfaces); infra supplies adapters; a composition root wires them; the presentation shell renders. Cross-module imports cross only through each module's index.ts barrel. The skill-analysis seam is the spine — new capability ≈ a new renderer, not a new pipeline — and the model gateway is the single metered entry to the model that every model call (build loop + evaluation) routes through.

Extend here first. A new capability is almost always defineCapability({ analyzer, renderers }) (analysis) or defineEvaluation({ evaluator, renderers }) (evaluation) on this seam — not a new service. Classify it before building: analysis gets an Analyzer and runs offline; evaluation gets an Evaluator, is handed the model gateway, and persists its result. The IR's source-spans are the contract the later React-Flow canvas + point-and-annotate reuse for free. ArtifactKind (the closed union of kinds) and Insight live here in the seam, not the kernel.

Shared kernel

src/shared — cross-cutting primitives, no domain knowledge, depends on nothing of ours
Result<T,E> · ok/err branded ids (SkillId, UserId…) DomainError (closed union) · domainError · notConfigured SseEvent · encodeSse
DomainError is a closed discriminated union — callers switch on tag exhaustively. Known tags: not_configured · not_found · persistence_failed · auth_failed · model_unavailable · cap_reached · seam_analyze_failed. Add a tag in src/shared/errors.ts; free strings are a compile error.

ArtifactKind and Insight are not here — they live in the skill-analysis seam (closer to the contract they shape), not the kernel.

Two rules carry the whole design.  1) New capability → a renderer on the seam (defineCapability), asked "what renderer is this?" before "what service?".  2) New external service → a port in the domain module + an adapter in infra, wired in container.ts. Everything else follows.
